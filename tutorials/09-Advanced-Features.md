# Advanced Features

Explore Ideatr's advanced features for deeper insights and analysis.

## Cluster Analysis

**What it does:** Deep dive into idea clusters to understand relationships and patterns.

**When to use:**
- You want to understand idea relationships
- You're looking for patterns
- You want to explore clusters in detail
- You're analyzing your idea collection

**How to use:**
1. Open Dashboard or Graph View
2. Use **"Analyze Idea Cluster"** command
3. Select cluster or idea to analyze
4. Review cluster analysis results:
   - Cluster members
   - Relationships
   - Common themes
   - Patterns

**What it shows:**
- Cluster composition
- Relationship strengths
- Common categories and tags
- Cluster statistics

## Tenuous Links

**What it does:** Discovers unexpected connections between ideas that might not be obviously related.

**When to use:**
- You want to find creative connections
- You're looking for inspiration
- You want to discover new angles
- You're exploring idea relationships

**How to use:**
1. Use **"Find Tenuous Links"** command
2. Review discovered links
3. Explore connections
4. Decide whether to:
   - Link ideas
   - Create combined idea
   - Explore further

**What it finds:**
- Weak but interesting connections
- Unexpected relationships
- Creative associations
- Novel connections

## Weekly Digest

**What it does:** Automated resurfacing of old ideas to prevent forgetting.

**When to use:**
- You want to review old ideas regularly
- You want to prevent idea loss
- You want automated reminders
- You want to stay aware of all ideas

**How to use:**
1. Enable in **Settings → Ideatr → Resurfacing Settings**
2. Configure frequency (daily, weekly, manual)
3. Configure time (if scheduled)
4. Review digest when generated

**What it includes:**
- Ideas older than threshold (default: 7 days)
- Summary of ideas (if enabled)
- Links to idea files
- Resurfacing recommendations

**Settings:**
- Enable/disable resurfacing
- Configure threshold (days)
- Configure frequency (daily, weekly, manual)
- Configure time (if scheduled)
- Enable summary generation
- Configure max ideas per digest

## Idea Statistics

**What it does:** Overview of your idea collection with counts, categories, and trends.

**When to use:**
- You want to understand your idea collection
- You're analyzing trends
- You want to see category distribution
- You're doing a collection audit

**How to use:**
1. Use **"Show Idea Statistics"** command
2. Review statistics modal:
   - Total ideas
   - By category
   - By status
   - By date
   - Trends

**What it shows:**
- Total idea count
- Category distribution
- Status distribution
- Date trends
- Tag frequency

## Export and Import

### Export Ideas

**What it does:** Export ideas to JSON or CSV format.

**When to use:**
- You want to backup ideas
- You want to analyze in external tools
- You want to share ideas
- You're migrating data

**How to use:**
1. Use **"Export Ideas"** command
2. Choose format (JSON or CSV)
3. Choose location
4. Export file is created

**What it exports:**
- Idea content
- Frontmatter metadata
- All fields and data

### Import Ideas

**What it does:** Import ideas from external sources.

**When to use:**
- You're migrating from another system
- You're importing from backup
- You're bulk importing ideas
- You're restoring data

**How to use:**
1. Use **"Import Ideas"** command
2. Select import file
3. Review import preview
4. Confirm import

**What it imports:**
- Idea content
- Frontmatter metadata
- All fields and data

## Error Logging

**What it does:** Built-in error tracking for debugging and bug reports.

**When to use:**
- You're experiencing errors
- You want to report bugs
- You're debugging issues
- You want error history

**How to use:**
1. Enable in **Settings → Ideatr → Error Logging**
2. Errors are automatically logged
3. View errors in settings
4. Export error logs for bug reports

**Settings:**
- Enable/disable error logging
- Configure max entries
- Configure retention days
- Export error logs

## Debug Mode

**What it does:** Developer-friendly debug logging.

**When to use:**
- You're debugging issues
- You want detailed logs
- You're developing features
- You want verbose output

**How to use:**
1. Enable in **Settings → Ideatr → General Settings**
2. Debug logs appear in console
3. More detailed error messages
4. Verbose operation logs
